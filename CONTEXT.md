# 建造信息分析

基于 BIM 模型数据的 AI 分析系统。用户通过前端提问，系统从 MongoDB 查询模型属性数据，经 AI（DeepSeek）总结后返回制式化分析结果。

## Language

**模型**:
一个从 RVT 或 IFC 文件解析入库的 BIM 数据单元。一个建筑体（如"万达广场1号楼"）可能由一到多个模型组成。前端指定分析范围时传入模型 `guid` 列表。
数据存储于 MongoDB `xEntity` 集合（另有 `xLevel` 楼层集合、`xBuilding` 建筑集合作关联查询）。
_Avoid_: 文件、文档、BIM 实体

**建造信息**:
模型的属性数据，存储于 MongoDB 中。业务属性集中在 `xEntity.propertySet` 数组，通过 `paramGroupId` 分组（如"工程量信息"）。包括空间位置、进度节点、成本数据、工程量数据、模版和脚手架周转数据等。
_Avoid_: 施工数据、工程数据

**进度节点**:
模型属性中的一个字段，描述该模型所处位置当前的施工进度阶段。值是语义化的阶段名称（如"基础施工"、"主体结构封顶"），人可读且直观。通过查询 `xEntity.propertySet` 中进度相关分组获取。
_Avoid_: 状态、阶段

**建造信息分析**:
一种制式化的分析输出，按固定维度组织：空间位置与进度节点、成本分析、工程量分析、模版和脚手架周转分析。附带若干快捷提问按钮。
_Avoid_: 报告、汇报
